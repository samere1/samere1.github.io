I developed a FUSE file system driver in C that mounts a 1MB disk image as a file system, based on the first ext file system for Linux. While the file system is mounted, all file system operations are handled by the FUSE program, such as creating, deleting, and listing files and (nested) directories, reading and writing from/to small files, creating hard links, and renaming files. It uses a superblock with an inode and data bitmap to keep track of free inodes and pages. Each inode keeps track of one file or directory by pointing to a page that holds data for the file or directory that it represents. It also holds other relevant information (size, reference count, etc.) and metadata.
